Snaposaurus

A game born in 2 days

During Ludum Dare 56

About it

Alongside other unexperienced members, we had to create a game 'from scratch' in a very short period of time. The theme of this Ludum Dare 56 was 'Collectibles', so we decided to create a game about a child who dreams of dinosaurs. He has to collect every dinosaur pictures he can find.

We've chosen to use Unity as our game engine, and thus, C# as our programming language.

How it went

Key steps

  • Phase 1 : Conceptualization of the game, moodboards, common setup of technologies.
  • Phase 2 : Development and design.
  • Phase 3 : Bug fix and production launch.

Challenges encountered

  • The tight deadline.
  • The learning of a new programming language (C#).
  • The first time working with git.
  • The coordination of people, especially between developer and designer.

Results

Despite the tight schedule and the inexperience of some team members, we managed to create a functional and enjoyable game within the 48-hour timeframe. The game was well-received by the Ludum Dare community, earning positive feedback for its creativity, music and overall atmosphere.

This experience taught us valuable lessons in teamwork, time management, and rapid prototyping, which we can apply to future projects.
